代理模式(Proxy)

发布 : 2017-01-16 分类 : 设计模式 浏览 :

意图

代理模式（Proxy），为其他对象提供一种代理，以控制这个对象的访问。1

适用性

  1. 远程代理（Remote Proxy）2
  2. 虚代理（Virtual Proxy）3
  3. 保护代理（Protection Proxy）4
  4. 智能引用（Smart Reference）5

￼

结构说明

Subject 类是 RealSubject 和 Proxy 的公共接口，这样可以在任何使用 RealSubject 的地方都可以使用 Proxy6

  1. 摘自《设计模式》 ↩
  2. 代理模式使得客户端可以访问远程主机上的对象，如同“中间人”的角色，在服务器和客户端之间进行数据的互相传递。 ↩
  3. 对一些占用系统资源较多或加载时间较长的对象，可以使用虚代理。在真实对象创建之前虚代理扮演替身，当真实对象创建之后虚拟代理将用户请求转发给真实代理。 ↩
  4. 保护开立可以控制对象的访问，用于对象应该有不同的访问权限的时候。(摘自：《设计模式》) ↩
  5. 智能指针 ↩

发布 : 2017-08-12 分类 : 如何自学成为一名软件工程师 浏览 :

　　你好，欢迎你的到来。本篇文章是 《如何自学成为一名软件工程师》系列的第一篇文章，本系列文章主要帮助那些没有基础，而又想通过自学成为一名软件工程师的朋友们。本系列所有文章中有关技术方面的都将围绕着 Linux 下 C/C++进行讲解。如果你凑巧想成为一名 C/C++ 工程师那这些文章也许会帮助到你。

我为何选择成为一名软件工程师

　　之所以选择做软件工程师，最大的原因也许就是每次解决问题时的成就感，当第一个Hello World输出后，当困扰已经的Bug解决后，这样一个个的小成功让我在其中找到了自信和无限的乐趣。
　　身边很多人说我很有毅力，坚持不懈的学习才能走到这一步。其实并不是这样，我只是明白学习编程对我的重要性，我明白只有这样才可以改变自己的命运，才可以离自己的目标更近一步。很多时候放弃并不是我们没有选择，而是我们选择太多… … 　　

自学编程学什么怎么学

　　在众多语言间我之所以选择 C/C++ 是因为它更接近底层，也许之后我会转向更高级的语言，但是学会C++还有什么语言能难倒我？
　　我的学习路线： 《C++ Primer》 –>《鸟哥的Linux 私房菜》–>《Linux 程序设计》–>《Unix网络编程》–>《数据结构与算法分析》–>SQL–>《HTTP权威指南》
　　我的学习过程是：首先直接过了一遍《C++ Primer》 ，之后又去培训机构学习了几个月，连同之前学习 汇编语言差不多用了2年的时间。在这个过程中 读 《C++ Primer》 是最痛苦的，而在培训机构最大的收获就是知道了学习的路线。
　　《鸟哥的Linux 私房菜》、《Linux 程序设计》、《Unix网络编程》、《数据结构与算法分析》、《HTTP权威指南》
这几本书我是在工作后读的。要注意不是这几本书不重要，而是应为我在培训机构中都对书中讲的知识重点做了培训。当然在之后的文章中我会详细的说明如何学习，学习什么。 　　

学到的东西能做什么？

　　如果你将上面推荐书籍中的知识都掌握了，那么恭喜你，你什么都做不了。是的，因为你缺乏实际开发经验。 
　　如果你将上面推荐书籍中的知识都掌握了，那么恭喜你，你算是刚刚入门，是的刚刚入门。之后的实际开发中你也许会用到 FCGI，MySQL，SQLServer，MongoDB等等很多，这些都是需要你在工作中慢慢积累的，此时的你可以做一份漂亮的简历，出去找工作了。 　　

自学没人指导怎么办？

　　很简单，Google，论坛，QQ群等等，只要努力一定会有人帮你解答你的疑问。当然在之后的文章中我会将自己遇到的问题分享出来，供你参考。 　　

怎么确定方向？

　　我了解的C++发展方向有 Linux或Windows两个平台的开发。Linux主要是服务器端的开发，Windows主要是客户端的开发。我的选择是 Linux，原因嘛！就是喜欢。其实你可以都学习，只不过是多了解一些API 罢了。 　　

最后

　　今天就说这么多，之后我陆续的更新该系列的教程。欢迎你持续关注 本博客。

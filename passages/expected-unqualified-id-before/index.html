mongodb宏重复

发布 : 2018-09-21 分类 : Bug笔记 浏览 :

错误描述

在使用MongDB C++ API 开发中出现如下错误，经排查发现为头文件包含位置错误。

错误提示

/home/lojze/local/include/mongo/util/goodies.h: At global scope:
/home/lojze/local/include/mongo/util/goodies.h:119: 错误：expected unqualified-id before ‘;’ token
/home/lojze/local/include/mongo/util/goodies.h:119: 错误：‘type name’ 的声明指定了两个以上的数据类型
/home/lojze/local/include/mongo/util/goodies.h：119：不能从先前的错误中恢复，退出

报错环境

系统

线程模型

gcc 版本

实现功能

CentOS 5.8 64Bit

posix

4.1.2 20080704 (Red Hat 4.1.2-55)

使用mongodb提取数据

解决方法

  1. 在包含mongo头文件时，该头文件必须先包含，包含mongo头文件的头文件也必须首先包含！
